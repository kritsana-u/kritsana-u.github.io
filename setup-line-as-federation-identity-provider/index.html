Setup LINE as Federation Identity Provider of AWS Cognito User Pool

Kritsana Uttamang | 05 Dec 2019 | 2 min read

Step 1 – Create OpenID Connect

  • Sign in to AWS account and go to Cognito service
  • Choose the user pool that you need to allow to sign in with LINE
  • On the left panel, choose Identity providers under Federation and choose OpenID Connect on the right. The OpenID Connect form will appear.
  • Enter LINE as the Provider
  • Enter Client ID and Client secret obtained from LINE
  • Choose Get as Attributes request method
  • Enter profile openid email as Authorize scope
  • Enter https://access.line.me as Issuer and leave Identifiers empty
  • Click Run discovery button on the right of Issuer and you will see that the discovery returns no result and there are additional fields show up to you. Don't worry. We just need to fill them up manually.
  • Enter https://access.line.me/oauth2/v2.1/authorize as Authorization endpoint
  • Enter https://api.line.me/oauth2/v2.1/token as Token endpoint
  • Enter https://api.line.me/v2/profile as Userinfo endpoint
  • Enter https://api.line.me/oauth2/v2.1/certs as Jwks uri
  • Click Create provider button

Step 2 – Setup Attribute Mapping

  • On the left panel, choose Attribute mapping under Federation and choose OIDC tab
  • Select LINE in the drop list of OIDC tab.

What's next?

Here I just show the step of setting up LINE as federation identity provider of the AWS Cognito user pool. You need to set up App integration to allow the application with the App Client ID to sign in with LINE.
